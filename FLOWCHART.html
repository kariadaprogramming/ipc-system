✅ Approval Workflow

2-Level Approval System

flowchart TD
    A[Data Input] -->|Submit| B[Status: Pending]
    
    B --> C{Approval Level 1}
    C -->|Pembina/Guru| D[Review Data]
    C -->|Auto-approve| E[Skip to Level 2]
    
    D --> F{Decision}
    F -->|Approve| G[Status: Pembina Approved]
    F -->|Reject| H[Status: Rejected]
    F -->|Need Review| B
    
    G --> I{Approval Level 2}
    I -->|Superadmin| J[Final Review]
    I -->|Auto-approve| K[Auto Approve]
    
    J --> L{Final Decision}
    L -->|Approve| M[Status: Approved]
    L -->|Reject| N[Status: Rejected]
    
    K --> M
    M --> O[Apply Changes]
    O --> P[Update IPC]
    P --> Q[Create History]
    Q --> R[Notify User]
    
    H --> S[Notify with Reason]
    N --> S
Approval Timeline:
  • Level 1 (Pembina): Immediate or within 24 hours
  • Level 2 (Superadmin): Within 48 hours
  • Auto-approval: For trusted users or low-point changes
  • Rejected requests can be resubmitted with corrections

📈 IPC Calculation Flow

flowchart TD
    Start([Base IPC]) --> Base[80 Points]
    
    Base --> Input{Input Types}
    
    Input -->|Prestasi| P[Calculate Points]
    Input -->|Organisasi| O[Calculate Points]
    Input -->|Event| E[Calculate Points]
    Input -->|Pelanggaran| Pel[Calculate Points]
    Input -->|Perilaku| Per[Calculate Points]
    
    P --> Sum[Sum All Points]
    O --> Sum
    E --> Sum
    Pel --> Sum
    Per --> Sum
    
    Sum --> Total[Total IPC]
    Total --> Check{Range Check}
    
    Check -->|< 0| Min[Set to 0]
    Check -->|> 100| Max[Set to 100]
    Check -->|0-100| Valid[Valid IPC]
    
    Min --> Save
    Max --> Save
    Valid --> Save[Save to Database]
    
    Save --> Log[Create History Log]
    Log --> Notify[Send Notification]
    Notify --> Update[Update Leaderboard]
IPC Range: 0 - 100 points
Base Score: 80 points
Minimum: 0 (floor)
Maximum: 100 (ceiling)

🗄️ Database Schema Overview

erDiagram
    USERS ||--o{ PRESTASI : has
    USERS ||--o{ ORGANISASI : has
    USERS ||--o{ EVENT : has
    USERS ||--o{ PELANGGARAN : has
    USERS ||--o{ PERILAKU : has
    USERS ||--o{ IPC_HISTORY : tracks
    USERS ||--o{ APPROVALS : submits
    USERS ||--o{ PERMISSIONS : has
    
    USERS {
        int id PK
        string nama
        string nis
        string nisn
        string nip
        string password
        enum role
        string kelas
        enum jurusan
        string grha
        int ipc_total
        int ipc_awal
        string foto
    }
    
    PRESTASI {
        int id PK
        int user_id FK
        string nama_lomba
        enum jenis
        enum tingkat
        int peringkat
        int point
        string foto
        enum status
    }
    
    IPC_HISTORY {
        int id PK
        int user_id FK
        int ipc_sebelum
        int ipc_sesudah
        string keterangan
        timestamp created_at
    }
    
    APPROVALS {
        int id PK
        int user_id FK
        enum jenis
        string data
        enum pembina_status
        enum superadmin_status
        text rejection_reason
    }
