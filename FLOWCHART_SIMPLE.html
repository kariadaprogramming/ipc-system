Flowchart Sederhana - Bahasa Indonesia

👑 SUPERADMIN

flowchart TD
    A([Login Superadmin]) --> B[Dashboard]
    B --> C{Pilih Menu}
    
    C -->|Kelola Akun| D[Kelola Akun]
    C -->|Wali Kelas| E[Wali Kelas]
    C -->|Approvals| F[Approvals V2]
    C -->|Input Access| G[Input Access]
    C -->|Laporan| H[Laporan Cetak]
    C -->|Logs| I[Logs]
    C -->|Profile| J[Profile]
    
    D --> D1[Buat Akun Guru]
    D --> D2[Buat Akun Siswa]
    D --> D3[Edit Akun]
    D --> D4[Hapus Akun]
    
    E --> E1[Assign Wali Kelas]
    E --> E2[Hapus Wali Kelas]
    
    F --> F1[Approve Data]
    F --> F2[Reject Data]
    
    G --> G1[Enable Input]
    G --> G2[Disable Input]
    
    H --> H1[Export Excel]
    H --> H2[Export PDF]
    H --> H3[Export Word]

📋 Fitur yang Bisa Diakses:

  • Kelola semua akun (Guru dan Siswa)
  • Assign Wali Kelas ke setiap kelas
  • Approve/Reject semua data yang masuk
  • Kontrol akses input data (Enable/Disable)
  • Cetak laporan (Excel, PDF, Word)
  • Lihat semua log aktivitas sistem
  • Edit profile sendiri

🔄 ALUR KERJA SISTEM

flowchart TD
    A([Mulai]) --> B{Login}
    B -->|Superadmin| C[Dashboard Superadmin]
    B -->|Guru| D[Dashboard Guru]
    B -->|Siswa| E[Dashboard Siswa]
    
    D --> F[Input Data Siswa]
    F --> G[Upload Bukti]
    G --> H[Submit]
    
    H --> I[Status: Pending]
    I --> J{Approval Pembina}
    
    J -->|Approve| K[Status: Pembina Approved]
    J -->|Reject| L[Status: Rejected]
    
    K --> M{Approval Superadmin}
    M -->|Approve| N[Status: Approved]
    M -->|Reject| L
    
    N --> O[Update IPC Siswa]
    O --> P[Buat Histori]
    P --> Q[Kirim Notifikasi ke Siswa]
    
    E --> R[Lihat IPC]
    R --> S[Lihat Notifikasi]
    
    L --> T[Kirim Alasan ke Guru]

📋 Penjelasan Alur:

  • Guru input data siswa → Status Pending
  • Pembina review → Approve/Reject
  • Superadmin review final → Approve/Reject
  • Jika Approved: IPC siswa di-update
  • Siswa menerima notifikasi perubahan
  • Siswa bisa lihat histori perubahan
